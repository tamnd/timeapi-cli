Each guide is built around a job rather than a command. They assume you have run
the [quick start](/getting-started/quick-start/).

- [Add a command](/guides/adding-a-command/) — model a real timeapi record
  and turn it into a command, a route, and a tool at once.
- [Resource URIs](/guides/resource-uris/) — use timeapi as a
  `database/sql`-style driver so a host can address timeapi as
  `timeapi://` URIs.

Add one guide per real task as you build out timeapi.
